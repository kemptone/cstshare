Contribute to CSTShare

Help grow the community by sharing your Logic Pro settings and knowledge

How to Contribute

1 Fork the Repository

Start by forking the project on GitHub to your own account.

git clone https://github.com/yourusername/audio-music-apps.git

2 Add Your Settings

Place your files in the appropriate category folder:

  • Channel Strip Settings/ - For .cst files
  • Plug-In Settings/ - For plugin presets
  • Patches/ - For instrument patches
  • Project Templates/ - For project templates

3 Name Your Files Clearly

Use descriptive names that indicate what the setting does. Good examples:

  • Warm Vocal.cst
  • Punchy Kick Drum.cst
  • 80s Synth Lead.patch

4 Commit and Push

git add .
git commit -m "Add warm vocal channel strip"
git push origin main

5 Create a Pull Request

Open a pull request on GitHub describing what you've added. The maintainers will review and merge your contribution.

Syncing Your Local Settings

You can work directly in your Logic Pro settings folder and use Git to sync your changes:

Initialize Git in your Audio Music Apps folder:

cd ~/Music/Audio\ Music\ Apps
git init
git remote add origin [your-repo-url]

Pull updates from the community:

git pull origin main

Push your new settings:

git add .
git commit -m "Add new settings"
git push origin main

Writing Blog Posts

Share your production tips and workflows by writing blog posts:

  1. Create a new markdown file in website/src/content/blog/
  2. Use the frontmatter format with title, date, author, and description
  3. Write your content in Markdown
  4. Submit via pull request

Contribution Guidelines

  • Only share settings you have the right to distribute
  • Use clear, descriptive names for files
  • Test your settings in Logic Pro before sharing
  • Include subcategories when appropriate (e.g., Bus, Track, Instrument)
  • Be respectful and helpful in discussions
  • Credit original creators when applicable
